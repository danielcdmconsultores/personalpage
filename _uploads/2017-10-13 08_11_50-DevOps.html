El mundo de DevOps está creciendo cada día y aunque no existe un ecosistema definido para DevOps, he listado un conjunto definido de herramientas.

Pero hay herramientas específicas de DevOps que se seleccionan e integran para definir los ecosistemas que cumplen con los requisitos del negocio.

Las herramientas de DevOps son en su mayoría de código abierto, pero también hay versiones empresariales.

Déjame agruparlas para que puedas entender los tipos de herramientas y sus usos.

Categorías de herramientas de DevOps:

  1. Colaboración
  2. Desarrollo
  3. SCM
  4. Gestión de la configuración
  5. Contenedores
  6. VMs
  7. Orquestación
  8. Servicios en la nube
  9. Proxy
  10. Construir
  11. CI / CD
  12. Análisis de código
  13. Pruebas
  14. Monitorización

1. Herramientas de Colaboración: Para comenzar con un proyecto ágil requeriremos algunas herramientas para la gestión de proyectos y lo más importante para la colaboración.

  • JIRA - Gran Herramienta para la planificación de proyectos (Scrum y Kanban), seguimiento de emisiones y colaboración en equipo. Pruébalo aquí.
  • Confluencia - Documentación del Proyecto y Sistema de Gestión del Conocimiento Empresarial. Pruébalo aquí.
  • HipChat - Herramienta para chat en grupo y colaboración. Pruébalo aquí.
  • Slack - Herramienta de colaboración de equipo con versiones gratuitas y empresariales.Pruébalo aquí.
  • Asana - Asana es la otra gran herramienta que puede ayudarle a colaborar con sus compañeros. Ofrece un sistema de gestión de proyectos inteligente, integrado y adaptable. Pruébalo aquí.
  • Trello - Trello es una emocionante herramienta de gestión de proyectos visual que puede organizar proyectos en tableros. Pruébalo aquí.

2. Herramientas de desarrollo : Los desarrolladores requiere algunas herramientas para iniciar la codificación. Dependiendo de la plataforma y el lenguaje de programación los desarrolladores tienen mucho para elegir. Algunas buenas herramientas son:

  • Eclipse : herramientas de desarrollo más utilizadas, ricas y confiables para desarrolladores Java. Tiene muchos complementos para elegir de su mercado que se puede integrar y poner en uso. Pruébalo aquí.
  • PyCharm - Herramienta rica para desarrolladores de Python. Pruébalo aquí.
  • RubyMine - herramienta rica para desarrolladores de Ruby. Pruébalo aquí ...
  • Notepad ++ - Una herramienta de propósito general para editar y codificar. Pruébalo aquí.
  • Atom - Herramienta rica de propósito general para edición y codificación. Pruébalo aquí.
  • Sublime text - Editor de texto enriquecido para la codificación. Pruébalo aquí.

3. Herramientas de gestión de código fuente (SCM): Los desarrolladores necesitan una herramienta SCM para el control de versiones. Las herramientas SCM desempeñan un papel clave en cualquier desarrollo y almacena y administra los códigos para los desarrolladores.

  • GIT - GIT es la herramienta SCM más utilizada por su naturaleza distribuida / descentralizada. Pruébalo aquí.
  • SVN - SVN es otra herramienta SCM más utilizada. Es una herramienta centralizada SCM. Pruébalo aquí.
  • GitHub - Servidor público Git alojado a través de Internet. Principalmente ampliamente utilizado para desarrollos de código abierto. Proporciona una interfaz de usuario web para GIT. Pruébalo aquí.
  • GitLab: otro Servidor Git alojado público accedido a través de Internet. Puede ser alojado en la premisa (en la red de la organización) también. Proporciona una interfaz de usuario web para GIT. Pruébalo aquí.
  • Gerrit - GIT Server de código abierto alojado en el local. Proporciona una interfaz de usuario web para GIT. Pruébalo aquí.
  • BitBucket - Un producto de Atlassian Enterprise para la solución GIT. Proporciona la interfaz de usuario web. Puede ser utilizado en la premisa o sobre el Internet público.Pruébalo aquí.

4. Configuración del servidor: A continuación, necesitamos proporcionar y configurar servidores para implementar, probar y liberar el producto desarrollado. Los Servidor son aprovisionados, configurados y mantenidos utilizando herramientas de administración de configuración.

  • Chef - Chef es una herramienta de gestión de configuración para automatizar la infraestructura. Administra servidores en la nube, en las instalaciones o en un entorno híbrido. Pruébalo aquí.
  • Puppet - Puppet es una herramienta de automatización de servidores de código abierto para la configuración y gestión. Funciona en sistemas Linux, Unix y Windows y realiza tareas administrativas (como agregar usuarios, instalar paquetes y actualizar configuraciones de servidor) basándose en una especificación centralizada. Pruébalo aquí.
  • Ansible - Ansible es el motor de automatización de TI de código abierto de Red Hat para el aprovisionamiento en nube, la gestión de configuración, el despliegue de aplicaciones, la orquestación intra-servicio y otras necesidades de TI en arquitecturas multinivel.Pruébalo aquí.
  • Saltstack - Salt Open es un software de gestión de la orquestación y la configuración para gestionar la infraestructura y las aplicaciones a escala. Es el proyecto de código abierto ascendente para SaltStack y se ejecuta en Linux y Windows. Pruébalo aquí.
  • CF Engine - CFEngine es en realidad mucho más antiguo que Puppet o Chef, que se remonta a 1993. Fue creado por Mark Burgess y, al igual que Puppet, comenzó como una herramienta de gestión de configuración de código abierto, no un producto de gestión de la configuración de la empresa. No se comercializó en 2008. CFEngine ha sido descrito como el abuelo de las herramientas de gestión de configuración. Pruébalo aquí.
  • Terraform - Terraform es una infraestructura como software de código de Hashicorp.Permite a los usuarios definir una infraestructura de centro de datos en un lenguaje de configuración de alto nivel, desde el que puede crear un plan de ejecución para construir la infraestructura en un proveedor de servicios como AWS. Pruébalo aquí.

5. Contenedores: Si usted no desea gastar mucho en la infraestructura, los contenedores son su mejor opción. Se ejecuta imágenes de sistema operativo y se puede girar hacia arriba y hacia abajo según su conveniencia. Lista de buenas soluciones de contenedores no son grandes a partir de ahora.

  • Docker - Docker es una plataforma abierta para que los desarrolladores y administrador de sistemas construyan, envíen y ejecuten aplicaciones distribuidas, ya sea en computadoras portátiles, máquinas virtuales de centros de datos o en la nube. Pruébalo aquí.
  • Packer - Packer es una herramienta gratuita y de código abierto para crear imágenes doradas para múltiples plataformas desde una única configuración de origen. Pruébalo aquí.

6. VMs portátiles: Los desarrolladores a menudo necesitan probar su aplicación en diferentes entornos y sistemas operativos. VMs portátiles son la mejor solución para tener múltiples entornos a su disposición.

  • Vagrant - Vagrant permite a los usuarios crear y configurar ambientes de desarrollo ligeros, reproducibles y portátiles. Pruébalo aquí.

7. Orquestación: Cuando se necesita un contenedor múltiple para interactuar unos con otros, se necesita un servicio de orquestación.

  • Docker Swarm - Docker Swarm es una herramienta de clustering y programación para contenedores Docker. Con Swarm, los administradores de TI y los desarrolladores pueden establecer y administrar un clúster de nodos Docker como un único sistema virtual. Pruébalo aquí.
  • Kubernetes - Kubernetes es una plataforma de código abierto diseñada para automatizar el despliegue, escalamiento y operación de contenedores de aplicaciones.Pruébalo aquí.
  • Juju - Juju es una herramienta de modelado de aplicaciones de código abierto.Implementar, configurar, escalar y operar su software en nubes públicas y privadas.Pruébalo aquí.

8. Servicios en la nube: Si no tenemos infraestructura de acogida de nuestra aplicación, no se preocupe! elegimos Nube. Los proveedores de Cloud son anfitriones de numerosos servicios en la nube que le ofrece múltiples opciones para utilizar los servicios alojados o para alojar su propia aplicación.

  • Amazon Web Services (AWS) - Amazon proporcionó el servicio en la nube. Pruébalo aquí.
  • Microsoft Azure - Microsoft proporcionó servicio en la nube. Pruébalo aquí.
  • Google Cloud Platform (GCP): Google proporcionó servicios en la nube. Pruébalo aquí.
  • Digital Ocean - Digital Ocean es otro proveedor de servicios en la nube popular. Pruébalo aquí.

9. Proxy & Load Balancing: Para mantener una comunicación fluida entre servicios agrupados en VMs y contenedores, siempre es bueno tener un servidor proxy y Load Balancer.

  • Nginx - NGINX es un servidor HTTP libre, de código abierto y de alto rendimiento y proxy inverso. Pruébalo aquí.
  • Apache Web Server - Apache Web Server es un software de creación, despliegue y administración de servidores web de código abierto. Inicialmente desarrollado por un grupo de programadores de software, ahora es mantenido por Apache Software Foundation. Pruébalo aquí.
  • HA Proxy - HAProxy es una solución gratuita, muy rápida y confiable que ofrece alta disponibilidad, equilibrio de carga y proxy para aplicaciones basadas en TCP y HTTP.Pruébalo aquí.

10. Build: Los desarrolladores necesitan construir su código localmente para comprobar cómo funciona antes de registrar el código en el servidor SCM. Las herramientas de construcción más famosas son:

  • Maven - Maven es una herramienta de gestión de proyectos y comprensión que proporciona a los desarrolladores un marco completo de ciclo de vida de la construcción. Pruébalo aquí.
  • Ant - Apache Ant es una herramienta de compilación basada en Java de Apache Software Foundation. Los archivos de compilación de Apache Ant están escritos en XML y se aprovechan de ser estándar abierto, portátil y fácil de entender. Pruébalo aquí.
  • Gradle - Gradle es un sistema de gestión de generación de propósito general de código abierto. Se basa en los depósitos ANT, Maven y Lvy. Es compatible con Groovy basado en lenguaje específico de dominio (DSL) sobre XML. Pruébalo aquí.

11 . Integración Continua / Despliegue Continuo: Con los Desarrolladores trabajando constantemente en diferentes servicios, la integración continua necesita estar en el lugar para integrar diferentes servicios juntos y construir con éxito la aplicación. Esto asegurará la detección temprana del error de integración y también implementará constantemente la aplicación en los entornos de destino.

  • Jenkins - Jenkins es un servidor de automatización de código abierto con un incomparable ecosistema de plugins para dar soporte. Es el más utilizado y muy popular por su integración con casi todas las herramientas de devops. Pruébalo aquí.
  • Travis - La fuerza principal de Travis es que es simple. A diferencia de Jenkins que permite plugins casi ilimitados, la creación de innumerables trabajos, flujos complicados, etc, Travis se basa en un único archivo .travis.yml que reside en la raíz de su código. Pruébalo aquí.
  • Bamboo - El bamboo es el producto de Atlassian. Es más fácil de usar y se ve mejor. La usabilidad es una de las primeras diferencias que uno notaría. Además, está bien integrado con el resto de productos de Atlassian (JIRA, Confluence, BitBucket, etc.).Pruébalo aquí.

12. Análisis de código: Para cualquier desarrollo, una cierta norma de codificación tiene que ser seguido. Además, un umbral necesita definir en qué medida la variación del estándar es aceptable. El análisis estático del código, necesita ser hecho para lograr esto. Esto proporciona una detección temprana y corrección en el ciclo de vida del desarrollo.

  • Sonarqube - Sonar es una plataforma de código abierto para la inspección continua de la calidad del código para realizar revisiones automáticas con análisis estático de código para detectar errores, olores de código y vulnerabilidades de seguridad en un lenguaje de programación de 20+. Pruébalo aquí.
  • PMD - PMD es un analizador de código fuente Java estático de código abierto que informa sobre el problema encontrado dentro del código de la aplicación. Pruébalo aquí.
  • CheckStyle - Checkstyle es una herramienta de desarrollo para ayudar a los programadores a escribir código Java que se adhiere a un estándar de codificación.Pruébalo aquí.
  • JUnit - JUnit es un framework de pruebas unitarias para el lenguaje de programación Java. Pruébalo aquí.

13. Pruebas: Una vez que la aplicación está lista y desplegada en los entornos de prueba, tiene que ser ampliamente probada. Varias pruebas son: Prueba de integración, Prueba de aceptación del usuario, Prueba de rendimiento, Prueba de carga, Prueba de penetración. Se pueden realizar ensayos manuales y automatizados. El proceso de DevOps utiliza las herramientas de pruebas automatizadas para hacerlo más rápido y sin errores. Es después de este paso cuando toda la aplicación pasa todas las barreras de prueba que se libera (desplegado a la producción).

  • Selenio - Selenio es un gran y probablemente el más popular herramienta de automatización de pruebas de código abierto. Automatiza las pruebas de todas las aplicaciones basadas en web y crea informes para el análisis. También proporciona un lenguaje de prueba de dominio específico (Selenese) para escribir pruebas en una serie de lenguajes de programación populares, incluyendo C #, Groovy, Java, Perl, PHP, Python, Ruby y Scala. Pruébalo aquí.
  • TestingWhiz - TestingWhiz es otra herramienta de automatización de pruebas, pero viene con un costo de licencia. Pruébalo aquí.

14. Monitorización: Por último, cuando la aplicación es en vivo y en tiempo real los usuarios están interactuando con ella algún monitoreo en tiempo real tiene que estar en su lugar para comprobar la aplicación, así como la salud del servidor. Este es un paso importante para entender que la aplicación se está comportando cuando está expuesta a los usuarios finales reales. Esto ayuda en la toma de entrada y el trabajo en mejoras adicionales.

  • Sensu - Sensu es un marco de monitoreo de código abierto para servicios de aplicaciones y sistemas. Puede recopilar y enviar métricas a una variedad de herramientas y proporcionar alertas para eventos definidos. Pruébalo aquí.
  • Nagios - Nagios es una aplicación de software libre y de código abierto que monitorea sistemas, redes e infraestructura. Nagios ofrece servicios de monitorización y alerta para servidores, conmutadores, aplicaciones y servicios. Alerta a los usuarios cuando las cosas salen mal y las alerta una segunda vez cuando el problema se ha resuelto.Pruébalo aquí.
  • NewRelic - New Relic APM es una solución de administración y monitoreo de aplicaciones que puede ayudar a los usuarios a monitorear aplicaciones que se ejecutan en ambientes híbridos, en la nube o en premisa. Pruébalo aquí.
  • DataDog - Datadog es una herramienta de monitoreo de SaaS dirigida a equipos de DevOps que toma datos de su aplicación y una amplia gama de otras herramientas y proporciona información y visualizaciones. Unifica los datos producidos desde su infraestructura y software en un solo lugar, lo que le permite crear paneles de control o buscar en los datos que lo alimenta. Pruébalo aquí.

Las herramientas de DevOps enumeradas anteriormente son sólo un conjunto de herramientas populares que se utilizan comúnmente. Cada herramienta tiene sus propios pros y contras y usted debe considerar todos los factores posibles antes de elegir nuestro propio conjunto de herramientas de la piscina.

Esperemos que esta publicación le ayude a comprender mejor las herramientas utilizadas en el ecosistema de DevOps.
